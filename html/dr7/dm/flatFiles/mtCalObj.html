MT Calibrated Objects

File Format: FITS binary table

Name: mtCalObj-sssssssss.fit, where sssssssss is the sequence number.

Produced by: mt, (excaliburPre), (kaliPre)

Used by: mtPost, excalibur, kali

Size: 14.4 Kb per file.

Archived? Yes

Description

Objects detected in the MT pipeline. Each file contains the object list for one MT sequence.

Primary Header



SIMPLE  =                    T
BITPIX  =                    8
NAXIS   =                    0
EXTEND  =                    T
SEQUENCE=                  XXX / MT sequence number.
QUALITY = 'XXX     '           / Data quality.

END

Frame Parameters Header

There is one HDU of this type. It contains some averaged photometric information (seeing and sky brightness) in each filter. There is one entry for each filter. 

XTENSION= 'BINTABLE'          
BITPIX  =                    8
NAXIS   =                    2
NAXIS1  =                   13
NAXIS2  =                  XXX
PCOUNT  =                    0
GCOUNT  =                    1
TFIELDS =                    4


TFORM1  = '1A      '          
TTYPE1  = 'FILTER  '           / Filter.
TFORM2  = '1E      '          
TTYPE2  = 'FWHM    '           / Median FWHM of brightest objects on the frame.
TUNIT2  = 'pixels  '          
TFORM3  = '1E      '          
TTYPE3  = 'SKY     '           / Global sky value for the frame.
TUNIT3  = 'counts/pixel'      
TFORM4  = '1E      '          
TTYPE4  = 'SKYERR  '           / Global sky value for the frame. (error)
TUNIT4  = 'counts/pixel'      

END

Object List Header

There is one HDU of this type. This contains the list of stars detected in this sequence. There is one entry for each star. 

XTENSION= 'BINTABLE'          
BITPIX  =                    8
NAXIS   =                    2
NAXIS1  =                   56
NAXIS2  =                  XXX
PCOUNT  =                    0
GCOUNT  =                    1
TFIELDS =                    6

FILTERS = 'u g r i z'          / Filter order in array fields.

TFORM1  = '1E      '          
TTYPE1  = 'LAMBDA  '           / lambda.
TUNIT1  = 'pixels  '          
TFORM2  = '1E      '          
TTYPE2  = 'LAMERR  '           / lambda error.
TUNIT2  = 'pixels  '          
TFORM3  = '1E      '          
TTYPE3  = 'ETA     '           / eta coord.
TUNIT3  = 'pixels  '          
TFORM4  = '1E      '          
TTYPE4  = 'ETAERR  '           / eta err.
TUNIT4  = 'pixels  '          
TFORM5  = '5E      '          
TTYPE5  = 'MAGS    '           / Sky-subtracted mags.
TUNIT5  = 'counts  '          
TFORM6  = '5E      '          
TTYPE6  = 'MAGSERR'          / Sky-subtracted mags. (error)
TUNIT6  = 'counts  '
