Tiling geometry files

The sdss_tiling_geometry files contain the information describing the geometry of each spherical polygon with respect to J2000 Equatorial coordinates, which sector it is a part of, and which tiling boundary region it is a part of. This allows you to determine the flux limit and the sampling fraction in each such region.

Two versions are given in two formats:

  1. sdss_tiling_geometry.ply the format used by Andrew Hamilton's Mangle software (the "polygon" format described on that page). Each polygon is specified by a block of lines of the form: 
    polygon                      0 ( 4 caps,     1.0000000 weight,   0.0000000000000000 str):
  0.0000000000000000   0.0000000000000000   1.0000000000000000   1.0219544779343330
 -0.9861304360790540   0.1659470297746836   0.0028890045854533  -0.0003381215107566
 -0.9811639210200757   0.1931769878587968  -0.0001070061362354  -0.0003381215107566
 -0.9840373658242012   0.1766602503941010  -0.0214853110859650   0.0000091016275849
  2. sdss_tiling_geometry.fitsa FITS format which contains the x and cm values for each cap (as defined by Mangle), as well as integers SECTOR and FIRSTBOUNDARY giving which sector number and which boundary number (the zero-indexed line in the tilingBoundary file). For reference, the header reads
