SDSS User Specification File

This page provides examples of files suitable for input on the SDSS Data Archive Server download page. You must create the appropriate file [instructions for doing so are on the (soon to be obsolete) sdssQT page] before selecting data products for download.

These are examples of *.csv files.

Example One:

run,rerun,camcol,field,filter
1356, 2, 3, 86, r
1356, 2, 3, 87, r
1356, 2, 3, 88, r

This file can be used to select fpC, fpBIN, fpM, fpObjc, fpFieldStat, fpAtlas, psFIeld, tsField, and tsObj files.
A similar file can select fpAtlasObj, but a column for object must be added as a final column.

Example Two:

plate,mjd,fiber,sprerun
513,51989,300,15
513,51989,312,15
513,51989,618,15
513,51989,638,15

This file can be used to select spMerge2d and spSpec files. It can also retrieve spPlate files, but in this case, the fiber column is superfluous.

This is an example of a *.par file.

typedef struct {
int run;
int rerun;
int camcol;
int field;
char filter[2];
} RRCFF

RRCFF 1356 2 3 86 r
RRCFF 1356 2 3 87 r
RRCFF 1356 2 3 88 r

This file is the same as the first .csv file, and used for the same data products, but it is in the .par file format. The complete description is at http://sdssdp7.fnal.gov/edoc/dm/flatFiles/yanny.html.

Here is an example of a *.tbl file

A *.tbl file can be generated by the (soon to be obsolete) sdssQT running on the OBJY database:

#XSTART ASCII
# 4
# "field" "segment.run" "segment.rerun" "segment.camCol"
# int int int int
# %d %d %d %d
#XEND
#QSTART
# select field,segment.run,segment.rerun,segment.camCol from Field where
# field>=300 && field < 305 && segment.run = 752
# ;
#QEND
